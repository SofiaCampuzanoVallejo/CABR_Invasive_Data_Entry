Invasive Species Data Entry Dashboard

This R Shiny application records field survey data and maintains an automated Excel database with monthly and quarterly summaries.

📁 Project Structure

Invasives_Data_Entry/
├── Invasive_Data_Entry.R    <-- Run this file
└── Data/
    └── invasive_species_data.xlsx  <-- Data saved here

🛠 Prerequisites

Ensure you have R and RStudio installed with these packages:

  • shiny
  • openxlsx

🚀 Getting Started

  1. Working Directory: Open the script. The setwd() command is configured for the Invasives_Data_Entry root folder.
  2. Launch: Click Run App in RStudio.
  3. Data Folder: The app will automatically create a /Data folder if it doesn't already exist.

⚠️ Validation Rule: Bag Count

To ensure data integrity, the app will not allow submission if the # of Bags field is 0. If you attempt to submit with 0 bags, an error message will appear and the data will not be saved.

📝 Using the Interface

Data Entry

  • Submission: Enter your site details, effort metrics, and species. Click Submit to append the data to the Excel master file.
  • Smart Reset: After submitting, most fields reset to default, but the Site dropdown stays on your last selection for faster repeat entries.
  • Undo: Use Remove Last Entry to delete the most recent row from the session and the Excel file simultaneously.

Data Viewing

  • Live Tables: View the last 5 entries, a full data log, and automated Monthly and Quarterly summaries directly in the dashboard.
  • Manual Access: Because the app saves in real-time, you can simply open the Excel file in the /Data folder to view your full formatted report.

🛑 Important Reminders

  • Excel Access: Close the Excel file before clicking Submit in the app. If the file is open in Excel, R will be blocked from saving and you will see a "Permission Denied" error.
  • Summaries: The Excel file contains three sheets: Data, Monthly_Summary, and Quarterly_Summary.
